# Issues To Fix

本文档用于持续记录 Web 端待修复与待增强需求，按“整体修改 + 各选项卡修改”组织。

## 1. 整体修改（跨页面/跨模块）

### 1.1 全站路径脱敏与展示边界统一
- **问题**：多个页面会暴露服务器绝对路径（如 `/home/...`），不适合终端用户。
- **要求**：
  - `GA/Model/Jobs` 等资源路径默认仅显示到 `workspace/` 相对层级。
  - 项目级路径最多显示到 `ChemDBWebVersion/`，不能再向上暴露。
  - 错误提示、详情面板、下载区域等都遵循同一规则。

### 1.2 表单状态持久化（会话内）
- **问题**：切换选项卡后表单输入丢失。
- **要求**：
  - 默认保留已填写内容（至少会话内）。
  - 返回页面自动恢复。
  - 提供显式“清空/重置”。

### 1.3 全站 SMILES 双击可视化
- **问题**：大量 SMILES 以纯文本展示，阅读成本高。
- **要求**：
  - 任意页面出现 SMILES 文本时，支持双击弹出 2D 结构图。
  - 解析失败给出明确提示。
  - 交互样式与错误提示全站一致。

### 1.4 训练超参数扫描（自动 sweep）能力建设（重要 + 危险，暂不实施）
- **优先级/风险**：**High / High Risk**
- **状态**：先立项，暂不实施。
- **目标**：支持参数范围配置并自动运行多组训练。
- **风险说明**：
  - 后续将涉及 `src` 与 `training` 多处超参数暴露与编排改造。
  - 资源成本、任务管理复杂度高，需先做参数分层与范围定义。

### 1.5 面向用户的“完整工作链路”显式化
- **问题**：当前用户需要自行理解流程阶段，难以判断当前处于哪一步、下一步该去哪里。
- **要求**：
  - 在产品层明确并可视化展示完整链路：
    1) 选择问题（Task）  
    2) 注入 GA 列表  
    3) 数据库构造  
    4) 模型样本抽样  
    5) 模型训练  
    6) 模型评估  
    7) 实现真实任务推荐
  - 每一阶段应有明确入口、状态与产物说明。
  - 用户可在任一阶段看到“上一阶段输出/下一阶段输入”的关联关系。

### 1.6 导航顺序按“单轮走通”设计
- **问题**：当前选项卡布局不完全按任务链路组织，用户需要反复翻找页面。
- **要求**：
  - 选项卡顺序应支持用户从上到下一次走完整流程。
  - 相邻页面应满足强上下游关系，减少跨页跳转成本。
  - 对流程关键节点提供“下一步去哪里”的页面内引导。

### 1.7 显式补入“模型样本重采样”步骤与页面入口（新增选项卡）
- **问题**：后端虽有 `training_data` 可调用步骤，但前端链路中没有“重采样”显式步骤，用户难以理解或触发。
- **要求**：
  - 在产品链路中显式加入“模型样本重采样（构造 `train/val/test_records.pkl`）”步骤。
  - 新增对应选项卡入口（或等价独立功能区），用于触发/查看重采样执行。
  - 页面需清楚说明输入依赖、输出产物与幂等行为（已有 split 时是否跳过）。
  - 提供安全重采样策略说明（例如需要清理旧 split/records 才会真正重采样）。

### 1.8 固定链路指示器（跨选项卡常驻）与影响范围高亮
- **问题**：用户切换选项卡时不知道自己处于链路哪个节点，也不清楚修改会影响哪些下游步骤。
- **要求**：
  - 在页面固定区域常驻显示完整工作链路（建议与 1.5 保持同一顺序与命名）。
  - 用户切换到某个选项卡时，点亮对应步骤与其下游步骤。
  - 明确提示“当前修改的影响范围”（例如会影响后续数据库构造/抽样/训练/评估/推荐）。
  - 该提示在各页面交互一致，避免每页单独定义不同口径。

## 2. 各选项卡修改

### a. Dashboard
- 暂无独立需求（可后续补充）。

### b. GA Library

#### b.1 GA set id 规则与提示优化
- **问题**：`ga_set_id` 规则严格（`^[a-z0-9][a-z0-9_-]{2,63}$`），且失败时提示不直观。
- **要求**：
  - 输入框旁展示命名规范与示例。
  - 报错返回可用格式与替代建议。
  - 评估允许大写并自动规范化为小写。

#### b.2 信息架构改造（Set -> Version 只读树）
- **问题**：层级感弱，定位 set/version 不直观。
- **要求**：
  - 提供只读树状浏览：`GA Set -> Version`。
  - 仅导航查看，不在树区直接修改。

#### b.3 预览改造（结构预览替代 CSV 主视图）
- **问题**：GA 本质是分子集合，CSV 文本预览不直观。
- **要求**：
  - 可滚动结构预览框，每页显示若干结构图 + `GA_ID`。
  - 支持分页、搜索、快速定位。

#### b.4 版本编辑能力（基于已有版本增删并另存）
- **问题**：当前主要依赖上传 CSV，难以做小幅增删。
- **要求**：
  - 支持“基于现有 set/version 创建新 version”。
  - 支持粘贴 SMILES 添加、选择性删除 GA。
  - 保持版本不可变：仅新增版本，不覆盖旧版。

### c. Task Library

#### c.1 Task Detail 信息降噪（面向用户语义）
- **问题**：Task Detail 当前展示了过多底层字段（如各类文件名/路径等），普通用户不关心且阅读负担重。
- **要求**：
  - 默认只展示用户关心的业务信息（任务名称/规模/状态/关键统计）。
  - 文件名、路径、内部字段放入“高级信息”折叠区，按需查看。
  - 字段命名与文案改为用户语义，减少工程术语直出。

#### c.2 Task 可视化预览（分子集合视角）
- **问题**：Task 本质也是分子集合，但当前展示方式缺少结构化可视化。
- **要求**：
  - 提供与 GA 类似的结构可视化预览区（结构图 + 关键标识）。
  - 支持可滚动浏览与分页，适配大规模集合。
  - 对 candidates/positives/negatives 提供清晰分区与数量提示，便于快速核对任务内容。

### d. Model Library

#### d.1 Model 资源树状结构（类 GA Library）
- **问题**：当前模型浏览缺少清晰层级，定位与对比不直观。
- **要求**：
  - 提供与 GA Library 类似的树状浏览结构（建议至少覆盖 `Project -> Run -> Model`）。
  - 树状区用于快速导航与选择模型详情，不承载高噪音技术字段。

#### d.2 模型详情以“超参数集合”作为主信息
- **问题**：当前容易把训练性能指标当作核心，但训练性能并不等同推荐性能。
- **要求**：
  - Model 详情优先展示该模型对应的超参数集合/版本与关键配置。
  - 训练性能指标不作为主展示信息，可降级到次级区域（折叠或次标签）。
  - 页面文案需明确“训练性能 ≠ 实际推荐性能”，避免用户误判。

### e. Model Selector Results

#### e.1 与 Recommendation 的信息架构关系重构
- **问题**：`Model Selector Results` 与 `Recommendation` 在用户心智上都属于“模型效果评估/筛选结果”，分离后容易造成入口割裂与结果查找困难。
- **方案建议**：
  - **优先方案（更符合直觉）**：合并为统一工作区（如“评估与推荐结果”），在同页用子标签区分 selector 与 recommendation。
  - **过渡方案（暂不合并）**：保留两个选项卡，但必须在导航中相邻，并统一结果浏览交互与术语。
- **验收标准**：
  - 用户能在单一连续流程中完成“筛选 -> 推荐 -> 查看结果”，无需跨远距离选项卡跳转。
  - 无论是否合并，两个模块的结果入口、筛选方式、下载方式保持一致。

### f. Jobs & Logs

#### f.1 默认信息分层与折叠
- **问题**：默认展示大量技术字段与日志尾部，噪音高。
- **要求**：
  - 默认只展示 `started_at`、`finished_at`、`status` 与“当前步骤进度”。
  - `job_id/job_type/command/full log` 放折叠面板，按需展开。
  - 日志预览默认折叠，用户点击后加载。

#### f.2 Step 级日志可视化入口
- **问题**：只能看 job 级日志，排查时还要手动去文件系统找 step log。
- **要求**：
  - 提供 Step Logs 视图（step 状态、开始/结束、退出码）。
  - 一键预览 `logs/<step>.log`（tail + 搜索）。
  - 可跳转对应 manifest，形成“状态-日志-产物”联动。

### g. Recommendation

#### g.1 多任务选择与综合评价
- **问题**：当前一次仅支持一个任务 + 一个模型。
- **要求**：
  - 支持多任务批量提交。
  - 增加综合评价配置（任务权重、汇总指标、排序策略）。
  - 同时展示逐任务结果与综合结果。

#### g.2 结果可发现性优化
- **问题**：结果入口不直观，用户常问“结果在哪”。
- **要求**：
  - 提交成功后显式给出本次结果入口与落盘位置。
  - 提供“本次提交结果”快捷卡片，而不是只靠历史目录手动查找。
  - 结果按任务/批次分组，可快速预览与下载。
  - 与 `Model Selector Results` 的导航关系保持连续（合并或相邻），避免用户在两个结果页间迷失。

### h. Hyperparameters

#### h.1 YAML 交互升级（依赖整体 1.4）
- **依赖**：超参数扫描能力落地后实施。
- **目标**：页面内编辑为主，上传 YAML 为辅。
- **要求**：
  - 显示当前版本 YAML 完整结构。
  - 支持页面直接修改并保存为新版本。
  - 自动命名新版本；上传入口保留为次选。
  - 结构与生效配置完全一致，保存前校验。

### i. Settings

#### i.1 导航位置调整
- `Settings` 移到最后一个选项卡。

#### i.2 SQLite Summary 展示优化
- **问题**：当前显示方式偏“机械字典”，不够可读。
- **要求**：
  - 改为卡片/表格化展示。
  - 每个指标提供 tooltip/help 注释。
  - 注释说明统计口径与业务含义。

### j. 新增选项卡：Workspace Manager（文件管理）

#### j.1 浏览与预览能力
- 显示文件树/分层列表，展示相对路径、创建/修改时间、大小、类型。
- 文本类可预览：`csv/yaml/yml/txt/md/json`。
- 二进制类（如 `.pt/.npz/.pkl`）显示“不可预览”。

#### j.2 管理员模式门禁
- 默认只读浏览。
- 点击“管理员模式”并输入密码后才可执行高风险操作（删除/修改）。
- 管理员模式不会因切换选项卡自动退出，必须手动退出。
- 默认密码需 SSH 上机修改，不在网页端改。

#### j.3 高风险操作约束
- 删除操作必须二次确认（含目标路径）。
- 提供成功/失败反馈与操作日志。
